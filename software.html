DrumLoop Sequencer

This is a PureData patch which uses a drum loop, cut into 16 pieces/steps. The tabel (top left) represents these steps. There is a function that changes one step randomly each time the pattern is looped. It also has some effects, most of which speak for themselves. The fiddle~ object is used to extract the pitch from the drum samples, which is used for the bass tones. "osc audio1/audio2" are FM synthesis effects.

AlgoComp

This project uses OpenFrameworks and PureData. The musical data is generated in OpenFrameworks and sent to PureData via OSC. The audio is synthesised in PureData. The user has the option of adding (and removing) objects into the world. There are three different types of objects: UserObjects, BeatObjects and RandomObjects.

The first UserObject, when added, will create a random melody, in a randomly selected scale. The other UserObjects, when added, will create a counterpoint melody on top of the first melody, fitting the musical scale of the current measure.

The BeatObjects provide a semi-randomly-created beat.

The RandomObjects, the small yellow ones, will randomly float from point to point.

When they collide with one of the UserObjects, a vibrato effect one the tone of that object will be triggered. The next note in that objects melody will also be randomised, so theres a good chance that note will no longer fit the current scale and harmony. This adds some dissonance.
